## 협업 방식

```
나 (오너)          Claude.ai (여기)        Claude Code (VS Code)
──────────────────────────────────────────────────────────
방향 제시     →   설계 + 명령어 작성   →   실행 + 빌드 + 배포
```

### 핵심 원칙
- 나는 방향만 이야기한다
- Claude.ai가 완성본 코드 직접 작성 → Claude Code에 전달
- 한글 명령보다 완성본 코드가 버그 적고 안정적
- 패치 3회 실패 → 파일 완전 새로 작성으로 전환

## Claude Code 명령어 패턴

### 파일 내용 확인
```
[파일명] 전체 내용을 채팅창에 텍스트로 붙여넣어줘. 코드블록으로.
```

### 파일 목록 확인
```
src 폴더 안에 [키워드] 포함된 tsx 파일 전체 내용을 채팅창에 텍스트로 붙여넣어줘.
```
> ⚠️ "경로"가 아니라 "내용"이라고 해야 채팅창에 출력됨

### git 히스토리 확인
```
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8; git log --date=short --pretty=format:"%ad %s" | Select-Object -First 100
```

### 표준 배포 명령어
```
Remove-Item -Recurse -Force .next
npm run build
git add . && git commit -m "메시지" && npx vercel --prod
완료되면 URL 한 줄 보고해줘.
```

```
└── AI가 회사 전체 운영 지원

🔗 연동
├── 개인 AI ↔ 회사 AI 연결
└── 개인 데이터는 개인 소유 유지
```

### AI 모델 전략
```
단기: Claude Haiku (저가 + 한국어 강함)
장기: 멀티 모델 (Haiku 기본 / Sonnet 복잡한 명령)
교체: aiService.ts 추상화 레이어로 한 줄 교체 가능
```

### 개발 순서
```
1단계: 히찌보드 완전 안정화 (지금)
2단계: 히찌보드 AI 채팅 통합 (실전 테스트)
3단계: Rehobot UI/UX 재구성 + 상용화
4단계: 개인/회사 2채널 연동
```
