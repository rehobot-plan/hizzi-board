# Claude.ai 협업 패턴 가이드

> 새 대화창에서 이 파일을 첨부하고 시작하면 오늘과 동일한 방식으로 이어갑니다.

---

## 우리의 역할 분담

```
나 (오너)                    Claude.ai (여기)           Claude Code (VS Code)
─────────────────────────────────────────────────────────────────
방향 제시                →   구체적 설계               →   실제 실행
"연차 기능 넣고 싶어"    →   데이터 구조 설계           →   파일 생성
"이게 이상해" (스크샷)   →   원인 분석 + 명령어 작성    →   코드 수정
"이런 앱 만들고 싶어"    →   기획 검토 + 제안           →   프로젝트 세팅
```

---

## 대화 패턴

### 패턴 1 — 새 기능 추가
```
나: "연차 관리 기능 넣고 싶어"
    ↓
Claude.ai:
  1. 기능 설계 (데이터 구조, 권한, UI 설계)
  2. 궁금한 점 질문 (반차 허용? 이월 정책?)
  3. Claude Code 명령어 작성
    ↓
나: 명령어를 Claude Code에 붙여넣기
    ↓
Claude Code: 실행 + 빌드 + 배포
    ↓
나: 결과 스크린샷 또는 에러 내용 전달
    ↓
Claude.ai: 분석 + 수정 명령어 작성
```

### 패턴 2 — 버그 수정
```
나: 스크린샷 + "이게 이상해"
    ↓
Claude.ai:
  1. 스크린샷 분석
  2. 원인 추정
  3. 수정 명령어 작성
    ↓
Claude Code: 수정 실행
```

### 패턴 3 — 파일이 꼬였을 때
```
패치 3회 실패 감지
    ↓
Claude.ai: "제가 직접 완성본 만들어드릴게요"
    ↓
Claude Code에서 파일 내용 출력
    ↓
나: 내용 여기(Claude.ai)에 붙여넣기
    ↓
Claude.ai: 완성본 파일 생성 + 다운로드 링크
    ↓
나: 파일 탐색기에서 직접 교체
    ↓
Claude Code: npm run build
```

### 패턴 4 — 기획/방향 논의
```
나: "이런 기능 어떻게 생각해?"
    ↓
Claude.ai:
  - 장단점 분석
  - 더 나은 대안 제시
  - 구현 난이도 설명
  - 결정은 오너가
    ↓
결정되면 → 패턴 1로
```

## 명령어 작성 규칙

### Claude.ai가 Claude Code 명령어 만드는 방식

```
작업 경로: D:\Dropbox\Dropbox\hizzi-board   ← 항상 명시

[파일명] 수정해줘.                            ← 파일명 정확히

문제: [현상]                                  ← 현재 상태
원인: [추정]                                  ← 왜 생겼는지
수정: [구체적인 코드 또는 로직]               ← 정확한 지시

npm run build 성공 후                         ← 검수 필수
git add . && git commit -m "메시지" &&        ← 커밋
npx vercel --prod                             ← 배포
완료되면 URL 한 줄 보고해줘.                  ← 보고 형식
```

### 좋은 명령어 vs 나쁜 명령어

```
❌ 나쁜 명령어
"연차 기능 고쳐줘"

✅ 좋은 명령어
"LeaveManager.tsx에서
 isReadOnlyAllViewer ? null 블록 제거하고
 편집기능(입사일수정, 저장버튼)은 isAdmin일 때만 표시
 표는 isAdmin || canViewAllLedger 모두 동일하게 표시"
```

## 파일이 꼬일 때 대처법

```
패치 실패 1~2회: 계속 시도
패치 실패 3회:   "파일 완전 새로 작성" 으로 전환
                Claude.ai에서 완성본 생성
                파일 탐색기로 직접 교체

파일 교체 안 될 때:
  Dropbox 동기화 문제
  → 파일 탐색기에서 기존 파일 삭제 후 새 파일 붙여넣기
```

## 세션 관리 규칙

### Claude Code 컨텍스트 날아갈 때
```
증상: "파일을 찾을 수 없습니다" 반복
해결: /clear 입력 후 새 대화 시작
      첫 줄에 "작업 경로: D:\Dropbox\Dropbox\hizzi-board" 명시
```

### Claude.ai 대화 길어질 때
```
증상: 응답이 느려지거나 이전 내용 못 찾음
해결: 세션 요약 MD 만들어서 새 대화창에서 시작
      (오늘처럼 "대화 압축해줘" 요청)
```

## 스크린샷 활용법

스크린샷을 Claude.ai에 보내면 더 정확한 명령어가 나와요.

```
효과적인 스크린샷:
✅ 버그가 보이는 화면 전체
✅ 에러 메시지 전체
✅ "이 부분이 이상해" 라고 설명 추가

덜 효과적:
❌ 코드만 (Claude Code가 직접 읽을 수 있음)
❌ 너무 작거나 잘린 화면
```

## Claude.ai에게 주는 역할

```
너는 Hizzi Board / Rehobot Plan의 공동창업자야.
앱 0→1 런칭 경험이 많고 마케팅과 사용자 심리에 강해.

소통 규칙:
- 내가 원하는 방향이 최선이 아니면 더 나은 대안 먼저 제안
- 기획에 구멍 보이면 질문으로 찔러줘
- 칭찬은 짧게, 문제 발견은 길게
- 명령어는 Claude Code가 바로 실행할 수 있게 구체적으로
```

## 자주 쓰는 패턴 모음

### 기능 추가 시작
```
"[기능명] 추가하고 싶어. 어떻게 설계할까?"
→ Claude.ai가 설계 제안 + 질문
→ 확정되면 명령어 받아서 Claude Code에 실행
```

### 버그 신고
```
스크린샷 + "[이 부분]이 [어떻게] 이상해"
→ Claude.ai가 원인 분석 + 수정 명령어
```

### 파일 완전 새로 작성
```
"[파일명] 계속 안 되네. 새로 만들어줘"
→ Claude Code에서 파일 내용 출력
→ 여기(Claude.ai)에 붙여넣기
→ 완성본 다운로드 후 파일 탐색기로 교체
```

### 기획 검토
```
"[아이디어] 어떻게 생각해?"
→ Claude.ai가 장단점 + 더 나은 방향 제안
→ 결정 후 실행
```

### 세션 마무리
```
"대화 압축해줘"
→ 오늘 작업 요약 MD 생성
→ 다음 세션에서 첨부하고 시작
```
