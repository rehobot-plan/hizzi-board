# Claude.ai 협업 패턴 가이드

> 새 대화창에서 이 파일을 첨부하고 시작하면 오늘과 동일한 방식으로 이어갑니다.

---

## 역할 분담

```
나 (오너)          Claude.ai (여기)        Claude Code (VS Code)
──────────────────────────────────────────────────────────
방향 제시     →   설계 + 명령어 작성   →   실행 + 빌드 + 배포
```

---

## ⭐ 작업 진행 원칙

### 작업 순서
```
에러 체크 필요한 작업 (빌드/배포/Firestore 배포) → 하나씩, 결과 확인 후 다음
파일 읽기 / 분석 / 코드 작성 → 한꺼번에 처리
Claude Code 명령어 → 항상 한 번에 실행 가능하게 묶어서 제공
```

### 파일 확인 방법 ⭐
```
📎 첨부 버튼으로 파일 직접 올리기
알파벳 순으로 요청 (찾기 쉽게)
2~3개씩 나눠서 올리기 (용량 문제 방지)
tsx / ts / md 파일 모두 첨부 가능
```

### 코드 수정 원칙
```
추측으로 명령 금지 → 파일 먼저 첨부해서 분석
수정 대상 + 연관 파일 구조까지 파악 (hizzi-master.md 파일 연관관계 섹션 참고)
any 타입 사용 금지 — 불가피 시 오너에게 이유 설명 후 승인
상태 변경 시 연계 데이터 함께 업데이트 (hizzi-master.md 흐름도 섹션 참고)
```

### 버그 반복 시
```
패치 2회 실패 → 연관 파일 전체 파악 후 재설계
패치 3회 실패 → 파일 완전 새로 작성
overflow 버그 → 2회 실패 시 바로 Portal 방식
```

### 매 작업 후 보고
```
✅ 완료: [방금 한 것]
🔄 진행 중: [지금 할 것]
📋 다음 순서: [우선순위 순]
```

## 공통 훅 (Custom Hook) 원칙 ⭐

### Claude의 제안 기준
```
같은 로직이 2개 이상 파일에 반복될 때
10줄 이상 useEffect/useState 패턴이 복붙될 때
앞으로 비슷한 기능이 추가될 가능성이 높을 때
→ "이 기능은 useXxx 훅으로 빼두면 앞으로 한 줄로 쓸 수 있어요. 만들까요?"
```

### 현재 적용된 훅
```
useEscClose(onClose, isOpen) — 모든 모달 ESC 닫기
위치: src/hooks/useEscClose.ts

사용법:
import { useEscClose } from '@/hooks/useEscClose';
useEscClose(() => setIsOpen(false), isOpen);
```

## 기술 부채 처리 원칙 ⭐

```
새 세션 시작 시 → 기술 부채 현황 먼저 확인 (hizzi-master.md 섹션 12)
새 기능 추가 전 → 우선순위 높은 부채 처리 제안
부채 처리는 별도 세션으로 잡기 (기능 개발과 섞지 않기)
```

---

## 상태 흐름 원칙 ⭐

```
새 기능에 상태 변경이 있으면 → 흐름도 먼저 정의
상태 변경 시 연계 효과 → Claude가 미리 파악하고 함께 처리
복잡한 연계는 hizzi-master.md 흐름도 섹션에 미리 정의
```

## 대화 패턴

### 패턴 1 — 새 기능 추가
```
나: "이 기능 넣고 싶어"
Claude.ai:
  1. 상태 흐름도 먼저 정의
  2. 공통 훅으로 뺄 부분 제안
  3. 기술 부채 영향 확인
  4. 명령어 작성 (한 번에 실행 가능하게)
```

### 패턴 2 — 버그 수정
```
나: 스크린샷 + "이게 이상해"
Claude.ai:
  1. 연관 파일 목록 확인 (master.md 참고)
  2. 파일 📎 첨부 요청 (알파벳 순)
  3. 코드 분석 → 원인 파악
  4. 수정 명령어 작성
```

### 패턴 3 — 세션 마무리
```
나: "오늘 마무리할게"
Claude.ai: MD 5개 업데이트본 생성 (다운로드 링크)
```

## 새 세션 시작 시 Claude에게 주는 역할

```
너는 내 공동창업자야.
히찌 패션 브랜드 사내 툴(히찌보드)을 개발 중이고
상용화 앱(Rehobot)도 준비 중이야.
글로벌로 갈 기업이라 처음부터 단단하게 구조를 잡아가고 있어.
속도보다 정확성이 우선이야.

첨부 파일들이 전체 맥락이야 (MD 5개 + 필요한 tsx 파일들).

핵심 규칙:
- 파일 확인은 📎 첨부로 (알파벳 순, 2~3개씩)
- 에러 체크 필요한 작업만 하나씩, 나머지는 묶어서
- any 타입 사용 금지 — 불가피 시 이유 설명 후 승인
- 공통 훅으로 빼면 좋은 부분 먼저 제안
- 상태 변경 시 연계 효과 미리 파악하고 함께 처리
- 기술 부채 우선순위 높은 것 먼저 제안
- 세션 마무리는 내가 제안할게
- 항상 일을 계속 진행시키는 태도 유지

오늘 할 작업: [여기에 입력]
```
