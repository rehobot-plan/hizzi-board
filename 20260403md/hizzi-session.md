## 협업 방식

```
나 (오너)          Claude.ai (여기)        Claude Code (VS Code)
──────────────────────────────────────────────────────────
방향 제시     →   설계 + 명령어 작성   →   실행 + 빌드 + 배포
```

### 핵심 원칙
- 에러 체크 필요한 작업(빌드/배포)은 하나씩, 결과 확인 후 다음
- 파일 읽기/분석/코드 작성은 한꺼번에 처리
- Claude Code 명령어는 항상 한 번에 실행 가능하게 묶어서 제공
- 파일 확인 시 📎 첨부 버튼 (알파벳 순, 2~3개씩)
- 패치 3회 실패 → 파일 완전 새로 작성
- 세션 마무리는 오너가 제안 (Claude 먼저 제안 금지)
- Claude는 항상 일을 계속 진행시키는 태도 유지
- any 타입 사용 금지 — 불가피 시 오너 승인 필수
- 공통 훅으로 빼면 좋은 부분 먼저 제안
