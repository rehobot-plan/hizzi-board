## 🔴 우선순위 1 — 진행 중

### 1. 에러 처리 통일
```
현황: 대부분 catch(e) { console.error(e) } → 사용자 피드백 없음
위험: 저장 실패해도 사용자가 모름

개선:
  모든 try-catch → addToast('오류 메시지') 추가

해당 파일:
  TodoItem.tsx       → handleEditSave catch
  CreatePost.tsx     → handleSubmit catch
  Calendar.tsx       → handleAdd, handleUpdate catch
  LeaveManager.tsx   → catch 전체
  todoRequestStore.ts → 각 함수 catch
```

### 2. any 타입 제거
```
현황: any 타입 곳곳에 남아있음
위험: 런타임에서 오류 발견

주요 위치:
  TodoItem.tsx       → const updates: any = {}
  CreatePost.tsx     → postData: any
  Calendar.tsx       → deleteConfirm target: any, ev: any
  todoRequestStore   → docData: any (불가피 여부 검토 필요)

개선:
  interface PostUpdates { content?: string; taskType?: 'work' | 'personal'; ... }
  interface CalendarEventUpdates { ... }
  Firestore 저장용은 Partial<T> 활용
```

## 🟡 우선순위 2 — 성장 대비

### 4. Firestore 저장 공통 함수화
```
현황: undefined 제거 로직이 각 함수마다 따로
개선:
  // src/lib/firestore.ts
  export function cleanFirestoreData<T extends object>(data: T): Partial<T> {
    const cleaned = { ...data };
    Object.keys(cleaned).forEach(key => {
      if ((cleaned as Record<string, unknown>)[key] === undefined)
        delete (cleaned as Record<string, unknown>)[key];
    });
    return cleaned;
  }
```
