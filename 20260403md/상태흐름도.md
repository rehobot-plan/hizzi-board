# Hizzi Board — 상태 흐름도

> Claude는 새 기능 설계 시 이 파일을 참고하여 연계 효과를 먼저 파악합니다.
> 새 상태 흐름이 생기면 반드시 이 파일에 추가하세요.

---

## 사용 원칙

```
새 기능에 상태 변경이 있으면 → 흐름도 먼저 정의 후 개발
상태 변경 시 연계 효과       → Claude가 미리 파악하고 함께 처리
흐름도에 없는 연계 발견 시   → 오너에게 먼저 보고 후 처리
```

## 1. todoRequest 상태 흐름

```
[pending] ──수락──→ [accepted]
    │                   │
    ├──반려──→ [rejected]│──완료처리──→ [completed]
    │                   │                   │
    └──취소──→ [cancelled]                  │
                        └────복구─────────←─┘
```

### 각 전환별 연계 처리

**pending → accepted (수락)**
```
1. todoRequests.status = 'accepted'
2. posts 컬렉션에 할일 생성
3. calendarEvents 생성 (dueDate 있을 때만)
   - 팀 요청이면 teamRequestId로 중복 체크 후 1개만 생성
연관 파일: todoRequestStore.ts > acceptRequest
```

**accepted → completed (완료처리)**
```
1. posts.completed = true, completedAt = new Date()
2. todoRequests.status = 'completed', resolvedAt = serverTimestamp()
3. 요청자 패널 요청함 "완료" 탭에 완료 알림 + 완료 시각 표시
연관 파일: TodoItem.tsx > handleComplete, todoRequestStore.ts > completeRequest
```

**completed → accepted (완료 취소/복구)**
```
1. posts.completed = false, completedAt = null
2. todoRequests.status = 'accepted', resolvedAt = null
연관 파일: CompletedTodo.tsx > 복구 버튼, todoRequestStore.ts > reactivateRequest
```

**pending → rejected (반려)**
```
1. todoRequests.status = 'rejected', rejectReason 저장
연관 파일: TodoRequestModal.tsx > handleReject
```

**pending → cancelled (요청자 취소)**
```
1. todoRequests.status = 'cancelled'
연관 파일: TodoRequestModal.tsx > cancelRequest
```

## 2. post (할일) 상태 흐름

```
[active] ──완료처리──→ [completed]
    │                       │
    └───────복구────────────┘
```

**일반 할일 완료**
```
체크박스 클릭 → posts.completed = true, completedAt = new Date()
연관: 없음
```

**요청받은 할일 완료 (업무 지시서 모달)**
```
완료처리 버튼 → posts.completed = true + todoRequests.status = 'completed' (반드시 함께)
연관: todoRequestStore.ts > completeRequest
```

**완료 복구**
```
✓ 버튼 → posts.completed = false
requestId 있으면 → todoRequests.status = 'accepted' (반드시 함께)
연관: todoRequestStore.ts > reactivateRequest
```

## 3. calendarEvent 상태 흐름

```
[일반 이벤트] → 수정/삭제 가능
[요청 이벤트] → requestId 연결됨
```

**달력 이벤트 색상 결정 흐름**
```
일정 추가 시:
  구분(업무/개인) + 공개범위(전체/나만/지정) 선택
  → getEventColor(taskType, visibility) 자동 결정

요청 수락 시:
  자동으로 color: '#993556' (핑크)
  taskType: 'work', visibility: 'all'
```

## 4. 연차(leaveEvent) 상태 흐름

```
[등록] → [confirmed: false] ──확정──→ [confirmed: true]
              │
              └── isPastOrToday = true → 수정/삭제 불가 (관리자 제외)
```

## 5. 앞으로 추가될 흐름 (예정)

```
[신청] → [pending] ──승인──→ [approved] → leaveEvent 생성
                  └──반려──→ [rejected]
```

```
3. 연계 처리 (함께 업데이트해야 할 것들)

```
