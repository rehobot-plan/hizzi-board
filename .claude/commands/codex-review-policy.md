## 검증 방법론 (R4.10)

PASS 판정 시 다음을 모두 충족해야 한다:

1. **acceptance criteria 매핑**
   - 명령 블록의 각 요구사항을 검증 항목으로 1:1 매핑
   - 누락된 요구사항이 있으면 즉시 FAIL

2. **실행 경로 trace**
   - 각 acceptance criteria마다 다음을 명시:
     - 입력 (어떤 데이터 / 어떤 상태)
     - 코드 경로 (어떤 함수 → 어떤 분기 → 어떤 출력)
     - 출력 (실제 렌더링 / 상태 변화)
   - 코드 라인 존재 확인만으로는 PASS 금지

3. **우회 해법 판정**
   - 각 수정 사항에 대해 "근본 해결 / 우회 / 임시방편" 분류
   - 우회·임시방편이면 PASS여도 명시적 경고

4. **미검증 경우의 수 플래그**
   - 단일 케이스 통과 시 "검증되지 않은 경우" 목록 출력
   - 예: "4/24~25 케이스만 검증. 4/24~26+겹침 / 동일날짜 단일 다수 미검증"

5. **시드 데이터 확인**
   - E2E 검증 시 시드 스크립트로 생성한 데이터 사용 여부 확인
   - 기존 DB 데이터 의존 시 경고
