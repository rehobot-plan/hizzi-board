## 기능 개요

**목적:** 직원 연차를 달력과 연동해서 체계적으로 관리
**기반:** 현행 근로기준법 연차 산정 방식 적용

---

## 권한 구조

```
관리자
├── 전체 연차 등록/수정/삭제
├── 직원 입사일 및 연차 기본 설정
├── 지난 날짜 확정 연차 수정 가능
└── 연차 열람자 지정 가능

연차 열람자 (관리자가 지정)
├── 지정된 직원 연차 대장 열람
└── 지난 날짜 확정 연차 수정 불가

본인 / 담당자
├── 미래 연차 등록 가능
├── 당일까지 수정/삭제 가능
└── 날짜 지난 확정 연차 수정 불가
```

## 연차 계산 방식 (근로기준법 기준)

### 1년 미만 신입
```
매월 개근 시 1일 발생
최대 11일 (입사 후 1년간)
```

### 1년 이상
```
1년 개근 시 15일 기본 발생
3년 이상부터 2년마다 1일 추가
최대 25일 한도

계산식:
기본 15일 + Math.floor((근속년수 - 1) / 2)
단, 최대 25일 초과 불가
```

### 예시
```
입사 1년차: 15일
입사 3년차: 16일
입사 5년차: 17일
입사 10년차: 19일
입사 20년차: 24일
입사 22년차~: 25일 (최대)
```

## Firestore 데이터 구조

### leaveSettings (직원 연차 설정) — 관리자만 입력
```typescript
{
  id: string;
  userId: string;           // 직원 uid
  userName: string;         // 직원 이름
  joinDate: string;         // 입사일 YYYY-MM-DD
  manualUsedDays: number;   // 관리자 직접 입력한 기사용 연차
  viewerEmails: string[];   // 열람 권한 부여된 이메일 목록
  createdAt: Date;
  updatedAt: Date;
}
```

### leaveEvents (연차 일정) — 본인/담당자 등록
```typescript
{
  id: string;
  userId: string;           // 신청자 uid
  userName: string;         // 직원 이름
  date: string;             // 연차 날짜 YYYY-MM-DD
  type: 'full' | 'half_am' | 'half_pm';  // 전체/오전반/오후반
  days: number;             // 1 또는 0.5
  memo?: string;
  confirmed: boolean;       // 날짜 지나면 자동 true
  createdAt: Date;
  createdBy: string;        // 등록한 사람 email
}
```

## 연차 계산 로직

```typescript
// 연차 발생 계산
function calcAnnualLeave(joinDate: string): number {
  const join = new Date(joinDate + 'T00:00:00');
  const today = new Date();
  const years = (today.getTime() - join.getTime()) / (1000 * 60 * 60 * 24 * 365);

  if (years < 1) {
    // 1년 미만: 매월 1일 (최대 11일)
    const months = Math.floor(years * 12);
    return Math.min(months, 11);
  }

  // 1년 이상: 15일 + 2년마다 1일 추가 (최대 25일)
  const extra = Math.floor((Math.floor(years) - 1) / 2);
  return Math.min(15 + extra, 25);
}

// 사용 연차 계산 (확정된 것만)
function calcUsedLeave(events: leaveEvent[], manualUsed: number): number {
  const today = new Date();
  const confirmedDays = events
    .filter(e => new Date(e.date + 'T00:00:00') <= today)
    .reduce((sum, e) => sum + e.days, 0);
  return manualUsed + confirmedDays;
}

// 잔여 연차
function calcRemainingLeave(joinDate: string, events: leaveEvent[], manualUsed: number): number {
  const total = calcAnnualLeave(joinDate);
  const used = calcUsedLeave(events, manualUsed);
  return Math.max(total - used, 0);
}
```

## UI 설계

### 달력 연차 등록 모달

```
연차 등록
─────────────────────
대상자: [직원 선택 드롭다운]
날짜:   2026-04-01
종류:   [전일] [오전반일] [오후반일]
메모:   (선택)
─────────────────────
         [취소]  [등록]
```

### 관리자 모드 — 연차 대장

```
연차 관리
─────────────────────────────────────────
[직원 선택 탭]
김진우 | 봉가 | 한다슬 | ...

─────────────────────────────────────────
김진우 연차 현황

입사일: 2023-03-01        [수정]
발생 연차: 17일
기사용 (수동입력): 3일    [수정]
확정 사용: 5일            (자동계산)
─────────────────────
총 사용: 8일
잔여 연차: 9일
─────────────────────

연차 사용 내역
──────────────────────────────────
날짜          종류    일수   메모      상태
2026-01-15   전일    1.0           확정 🔒
2026-02-20   오전    0.5           확정 🔒
2026-04-01   전일    1.0           예정
──────────────────────────────────
              [+ 연차 추가]
```

### 달력에서 연차 표시
```
날짜 셀에 연차 블락 표시
색상: 직원별 고유 색상 또는 연차 전용 색상
표시: "김진우 연차" 또는 "봉가 반차"
확정된 연차: 🔒 아이콘 표시
```

## 구현 순서

```
1단계: Firestore 구조 생성
   - leaveSettings 컬렉션
   - leaveEvents 컬렉션

2단계: 관리자 모드 — 직원 설정
   - 입사일 입력
   - 기사용 연차 수동 입력
   - 열람자 지정

3단계: 연차 등록 모달
   - 달력에서 날짜 클릭 시 연차 등록 옵션 추가
   - 대상자(직원) 선택
   - 전일/반일 선택

4단계: 날짜 지난 연차 자동 확정
   - 날짜 지나면 confirmed: true 자동 변경
   - 확정된 연차는 일반 사용자 수정 불가

5단계: 연차 대장 UI
   - 관리자 모드에 연차 관리 탭 추가
   - 발생/사용/잔여 계산 표시

6단계: 열람자 권한
   - 관리자가 지정한 사람만 연차 대장 열람 가능
   - 확정 연차 수정 권한은 없음

7단계: 달력 연동
   - 연차 이벤트를 달력에 표시
   - 🔒 확정 표시
```
