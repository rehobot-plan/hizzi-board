# HIZZI Board — 세션 운영

> 세션 경계 절차(시작·종료·프리셋)의 단일 출처.
> Claude.ai가 판단·제안을 주도하고 Claude Code가 기계적으로 실행한다.
> 세션 내 작업은 harness.md 참조. 수정: 오너 승인 필수.

## 0. 주체 분담

세션 경계에서 발생하는 운영 절차는 Claude.ai가 주도한다. 세션 내 코딩 작업(공장 안)과 달리, 세션 경계는 "무엇을 정리·이관할지" 판단이 핵심이라 기획자 역할이 먼저다.

- **Claude.ai** — 판단·제안. 오너와 합의된 결정을 Claude Code에 명령으로 전달.
- **Claude Code** — git 명령·파일 조작·`presets.json` 갱신 같은 기계적 실행.
- **오너** — 각 단계 제안에 "OK / 수정" 반응만.

이 파일은 Claude.ai가 자기 응답 구조에 참조하는 동시에 Claude Code가 세션 종료 실행에 참조하는 공통 레퍼런스다. 프리셋 상시 포함 파일이라 새 세션에서도 자동 로드된다.

## 1. 세션 시작

### 첨부 파일

새 대화방 시작 시 오너가 업로드하는 파일 (프리셋 시스템이 `md-presets/_staging/`에 미리 준비). **아래 번호 순서대로 읽는다** — 정체성과 구조가 먼저 세팅돼야 현황을 해석할 틀이 생긴다.

1. `CLAUDE.md` *(정체성·역할)*
2. `md/core/harness.md` *(프리셋 상시 포함, 공장 구조)*
3. `md/core/rules.md` *(공장 안 규칙)*
4. `md/core/session.md` *(프리셋 상시 포함, 세션 경계)*
5. `md/log/progress.md` *(현황)*

### 주입 확인 (첫 단계)

세션 시작 프롬프트를 받으면 Claude.ai는 먼저 다섯 파일이 실제 주입됐는지 내용으로 확인한다. 파일명만 보이고 내용이 비어있으면 로딩 실패다 — 과거 CLAUDE-detail.md가 크기 한계로 안 들어왔던 사례가 있음. 하나라도 빠지면 즉시 오너에게 보고하고 진행을 중단한다.

### 최초 응답 구조

확인 후 단일 응답으로 회신한다:

1. **직전 세션 인계** — progress.md 마지막 로그 한 문단 요약
2. **다음 TODO** — progress.md "다음 세션 1순위" 그대로 표시
3. **필요 MD 추가 로드** — CLAUDE.md 파일 지도 적용 결과
4. **오늘의 추천** — CLAUDE.md 제안 포맷 4줄
5. **오너 응답 대기**

오너가 "OK" 한 단어로 진입 가능해야 한다. "어느 게 좋을지 모르겠어서" 류 되묻기는 금지 — 판단은 Claude.ai가 하고 근거는 추천 이유에 쓴다. TODO가 비어있거나 항목 간 의존이 모호해 추천 자체가 불가능한 경우만 예외로 질의 허용.

## 2. 세션 종료 — 채팅방 이관 12단계

세션 종료 = 코드 종료가 아니라 대화방 이관. 핵심 셋: MD 업데이트 체크, progress.md 업데이트, 다음 세션 프리셋.

**발동 방식:** 오너가 자연어로 "세션 종료하자" / "정리하고 넘어가자" 류 발화. 슬래시 명령 `/close-session`은 폐기. 오너 발화 한 번을 시작점으로, Claude.ai가 12단계 중 오너 검수가 필요한 항목들(3·4·5·7·8·11)을 **한 응답으로 통합 제안**한다. 오너는 통합 제안 전체에 "ㅇㅇ" 한 마디 또는 "X번 수정" 지시로 반응. 개입은 시작점 1회 + 통합 검수 1회로 끝난다.

Claude.ai는 세션 중에 종료 시 꺼낼 항목(부채·학습·완료 TODO)을 암묵적으로 누적한다. 종료 시점에 갑자기 수집하지 않는다.

### 12단계

| 단계 | 내용 | Claude.ai 역할 | 실행 |
|:---:|---|---|:---:|
| 1 | 완료 작업 정리 | 주요 완료 시점 기억·정리 | Claude.ai (자동) |
| 2 | progress.md 읽기 | 내용 파싱 | Claude.ai (자동) |
| 3 | 완료 TODO 제거 | TODO ↔ 실제 파일 대조 | 통합 제안 포함 → Code |
| 4 | 현재상태 replace | 세션 변경 반영한 초안 (before/after) | 통합 제안 포함 → Code |
| 5 | 작업로그 세션 블록 흡수 | 세션 중 1-6이 남긴 짧은 한 줄들을 읽고 세션 단위 요약 블록 작성. 블록 추가 후 해당 한 줄들은 제거 (최종적으로 작업로그엔 세션 블록만 남음). **산출물 "수정 MD" 목록은 실행 완료분(커밋된 것)만 포함 — 기획 논의·계획만 된 항목은 "계획 but 미반영" 라인으로 분리 표기** | 통합 제안 포함 → Code |
| 6 | 작업로그 아카이브 | 완료 세션 3건 이상이면 최근 2건만 남기고 `md/archive/progress-YYYY-MM-{X}.md`로 이관 | Claude.ai 자동 판단 → Code |
| 7 | master-debt.md 갱신 | 부채·우회·known issue 감지 시 (없으면 "없음" 보고) | 통합 제안 포함 → Code |
| 8 | .harness/MEMORY.md 갱신 | 학습 패턴·실패 사례 감지 시 (없으면 "없음" 보고) | 통합 제안 포함 → Code |
| 9 | 세션 번호 검증 | 직전 + 1 = 이번 | Claude.ai (자동) |
| 10 | untracked 감사 | md/ 하위 untracked 확인 | Code (`git status`) 자동 → Claude.ai 해석 |
| 11 | 다음 세션 프리셋 | 1순위 TODO 기반 파일 분류 → `presets.json` 갱신 → **`프리셋` 트리거 실행 전 대상 MD 각각의 mtime·파일 말미 타임스탬프를 Code가 확인해 세션 말 기대 상태와 괴리 없음을 보고** → 트리거 실행 | 통합 제안 포함 → Code → 오너 실행 |
| 12 | session-started.flag 삭제 | `.harness/session-started.flag` 제거 | Code (자동) |

### 통합 제안 포맷

Claude.ai는 단계 1·2·6·9·10 자동 수행 결과를 먼저 정리한 뒤, 오너 검수가 필요한 3·4·5·7·8·11의 실행 명령 블록을 한 응답에 모아 제시한다. 실행 명령 블록은 반드시 파일 형태로 /mnt/user-data/outputs/에 산출해 첨부 — 오너가 별도 요청하지 않아도 기본 동작이다.

```
세션 종료 준비 완료.

자동 수행 결과:
- 세션 번호: #N (직전 + 1 = 이번, 검증 완료)
- untracked 감사: (결과)
- 아카이브 트리거: (해당 / 불필요)

검수 필요 항목:
[3] 완료 TODO 제거 — (대조 결과, 제거 대상 목록)
[4] 현재상태 replace — (before/after)
[5] 작업로그 세션 블록 — (신규 블록 전문)
[7] master-debt.md — (필요 / 없음)
[8] MEMORY.md — (필요 / 없음)
[11] 다음 세션 프리셋 — (파일 목록)

전체에 이상 없으면 "ㅇㅇ" 한 마디로 진행. 특정 항목만 수정하려면 "X번 수정: (내용)".
```

오너 승인 후 Code가 3~5, 7, 8, 11, 12를 순차 실행.

완료 시 Claude.ai가 "세션 종료 완료 ✅" 출력.

### MD 업데이트 체크 (단계 4·5 선행)

이번 세션에서 확정된 규칙·패턴·구조 변경이 어느 MD에 반영돼야 하는지 Claude.ai가 판단·제안. 대상 후보: `rules-detail` / `flows` / `master` / `patterns` / `ux-principles` / `CLAUDE` / `harness` / `session` / `principles`. 반영 대상 없으면 "없음".

### 제약

- 3·4·5·7·8·11은 통합 제안에 포함돼 오너 승인을 받는다. 승인 없이 실행 금지.
- 11은 **갱신 + 트리거 실행까지** PASS. 갱신만 하고 트리거 누락하면 다음 세션이 구버전으로 시작.
- 6은 자동 판단 (완료 세션 3건 이상 = 트리거).
- 11 생략: 오너가 "프리셋 생략" 발화한 경우만.
- CLAUDE.md [7] before/after 검수 규약이 MD 수정을 별도로 보장하므로, 세션 종료 시에도 단계 4 제안은 before/after 포맷을 따른다.
- **단계 5 산출물 "수정 MD" 목록은 Code가 git log·커밋 해시로 교차 검증한 실행 완료분만 포함한다. 기획 논의·계획만 된 항목은 "계획 but 미반영 → 다음 세션 이관"으로 분리 표기 (MEMORY #61-c·#62-a).**
- **단계 11 프리셋 복사 직전, Code는 대상 MD 각각의 mtime·파일 말미 타임스탬프를 확인해 세션 말 기대 상태와 괴리 없음을 보고한다. 괴리 감지 시 복사 중단·오너 보고 (MEMORY #62-d · #61 drift 사고 재발 방지).**

## 3. 프리셋 — 다음 세션 파일 준비

`프리셋` 자연어 트리거는 두 모드.

### 모드 A — 복사

**조건:** "프리셋" + "만들어" 계열 동사 없음.

`presets.json`의 `current.files` 배열을 읽어 `md-presets/_staging/`에 복사.

- 대상: `D:\Dropbox\Dropbox\hizzi-board\md-presets\presets.json`
- 복사처: `D:\Dropbox\Dropbox\hizzi-board\md-presets\_staging\`
- `_staging` 없으면 생성, 기존 파일 전량 삭제 후 신규 복사
- 완료 후 파일 목록 + 크기·타임스탬프 보고

예시: "프리셋", "프리셋 실행", "프리셋 복사"

### 모드 B — 생성+복사

**조건:** "프리셋" + "만들어" 계열 동사 포함.

직전 Claude.ai가 지정한 파일 세트로 `presets.json`을 갱신하고 같은 실행에서 `_staging` 복사까지 완료.

**파일 세트 출처 우선순위:**
1. 같은 프롬프트 안에 파일 목록 명시 → 그것 사용
2. 명시 없음 → 직전 Claude.ai 출력에서 추출
3. 둘 다 없음 → 오너 확인 질의 (예외)

**부가:**
- `name` / `description` / `updated` 필드 있으면 함께 갱신
- 커밋: `chore(presets): {name} 프리셋 갱신`

예시: "프리셋 만들어줘", "프리셋 갱신해줘"

### 공통

- **실행 주체:** Claude Code (PowerShell 프로필 의존 없음)
- **상시 포함 파일:** `md/core/harness.md` + `md/core/session.md` (라우팅·주제 무관 무조건 포함)
- **실패 처리:** `presets.json` 파싱 실패 / 지정 파일 미존재 시 중단 + 원인 보고, `_staging`은 건드리지 않음
