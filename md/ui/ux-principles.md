# 히찌보드 — UX 원칙

> 새 기능 설계 전 반드시 읽는다.
> 확정된 원칙이며 임의 변경 금지.
> 변경 필요 시 오너 승인 후 이 파일을 먼저 업데이트한다.

---

## U1. 정보 계층 구조

```
패널 (Panel)
  └─ 탭바: 패널명(좌측) + 탭(우측)
      └─ 할일 탭: 필터 바 (업무/요청/개인) + 게시물
      └─ 메모 탭: 필터 바 숨김 + 게시물만
```

---

## U2. 필터 원칙

```
버튼: 업무 / 요청 / 개인 (전체 버튼 없음)
기본: 업무 + 요청 동시 활성 (중복 선택 허용)
아무것도 선택 안 된 상태 → 전체 표시 (엣지케이스 방어)
메모 탭: 필터 바 완전 숨김
```

---

## U3. 할일 아이템 구조

```
[왼쪽 2px 컬러선] [체크박스] [별] [제목 + 휴지통(일반만)]
                                   [태그들] [dueDate시계] [날짜]

체크박스: 맨 왼쪽, 모든 할일 즉시 완료 (요청도 동일)
별:       opacity 0.25 → hover 0.6 → starred 1.0
휴지통:   일반 할일만, 아이콘만 (border/background/shadow 없음)

클릭 레이어: position absolute, left 66px (체크박스+별 영역 제외), zIndex 1
체크박스·별: position relative, zIndex 2
→ 체크박스/별/휴지통 클릭 시 stopPropagation으로 팝업 차단
```

## U4. 할일 데이터 구조

```typescript
// 현재: content 필드가 제목 역할

  dueDate?: string   // 기한 (YYYY-MM-DD)
```

## U5. 정렬 기준

```
할일 목록:
  1. starred = true → 최상단
  2. dueDate 임박순
  3. createdAt 최신순

메모 목록:
  1. starred = true → 최상단
  2. createdAt 최신순
```

## U6. 삭제 원칙

```
단건 삭제:
  좌←우 스와이프 → soft delete (deleted: true + deletedAt 타임스탬프)
  confirm 없음 / 3층 복구 동선 (1층 토스트 5초, 2층 24h 링크, 3층 기록 메뉴)
  복구 시 completed: false 함께 초기화
  기존 hover 휴지통 아이콘 제거 (mobile-first 프레이밍)

3층 복구 구조:
  1층 — 토스트 "삭제됨 · 실행 취소" (5초)
  2층 — 리스트 하단 "최근 삭제 N개 →" (24시간 창, 0건 시 숨김)
  3층 — 탭바 메뉴(···) → "기록" → [휴지통] 탭 (24h 경과분, 30일 후 hard delete)

팝업 내 삭제:
  삭제 버튼 → try/catch + addToast

요청 할일 스와이프 삭제:
  post soft delete + todoRequest.status = 'cancelled' (cascade 필수)
  1층·2층·3층 복구 시 todoRequest.status를 삭제 직전 상태(pending|accepted)로 cascade 복귀
  flows.md FLOW 1 참조

bulk 삭제:
  try/catch/finally 필수 (rules.md S2 참조)
```

## U7. 완료 처리 원칙

```
체크박스 클릭 (모든 할일 공통):
  체크박스 시각 18px · 터치 영역 44px (행 전체 높이)
  → 0.6s 애니메이션 후 완료
  → 요청 할일도 즉시 완료 (팝업 없음)
  → cascade: posts.completed = true + todoRequests.status = 'completed' + completedAt 타임스탬프

팝업 내 완료처리 버튼 (요청 할일):
  → 동일 cascade
  → 완료 후 팝업 닫힘

3층 회수 동선 (U6 삭제와 대칭):
  1층 — 토스트 "완료됨 · 되돌리기" (5초)
  2층 — 리스트 하단 "최근 완료 N개 →" (24시간 창, 0건 시 숨김)
  3층 — 탭바 메뉴(···) → "기록" → [완료] 탭 (24h 경과분, 영구 보관)

기존 CompletedTodo.tsx 하단 섹션 → 2층 링크 구조로 재편
  활성 리스트와 시각적 분리, 패널 과확장 원인 제거
```

## U8. 수정 원칙

```
모든 할일/메모:
  아이템 클릭 → 팝업에서 수정
  제목: 헤더 클릭 → 연필 아이콘 활성 → 인라인 편집
  내용/기한/첨부/구분/범위: 팝업 바디에서 수정
  ··· 메뉴 없음 (완전 제거)

요청 할일: 팝업에서 수정 가능 (저장 버튼)
```

## U9. 요청함 (TodoRequestModal) UX ⭐

```
위치: 패널 탭바 우측 우편함 아이콘
배지: 수락 대기 건수만 카운트

섹션 1 — 내가 수락해야 함: toEmail=나, status=pending,   color #993556
섹션 2 — 내가 진행 중:    toEmail=나, status=accepted,  color #C17B6B
섹션 3 — 상대방 대기 중:  fromEmail=나, pending|accepted, color #9E8880
섹션 4 — 완료·반려·취소:  기본 접힘,                    color #3B6D11
```

## U10. 완료 알림 원칙

```
트리거: 수신자가 요청 할일 완료 처리 시 (pending|accepted → completed 어느 경로에서든)
요청자: 봉투 배지 카운트 증가 + 모달 섹션 4 "완료·반려·취소" 내 시각 강조

배지 카운트 = pending 건수 + 미확인 완료 알림 건수
  "내가 액션해야 할 것 + 내가 궁금해할 만한 것" 두 축 합산

미확인 해제 조건:
  모달 오픈 시 해당 항목의 todoRequests.seenAt 타임스탬프 업데이트
  seenAt 이후 재진입 시 강조 제거

구현: onSnapshot에서 * → completed 변경 감지, fromEmail === 나 조건
```

## U11. 모바일 대응 원칙

```
프레이밍: 메인 6패널 그리드 = 모바일 1인 뷰의 데스크탑 미니 프리뷰
  패널 설계가 "내 모바일 화면이 데스크탑에서 어떻게 축소돼 다른 사람과 나란히 보이는가"에서 출발
  패널 max-height: min(600px, 70vh) — 모바일 viewport 아스펙트와 일관

터치 타겟: 최소 44px (체크박스 시각 18px + 행 전체 터치 영역 확장 방식)
hover 전용 UI: 금지 (스와이프·탭 등 mobile-native 상호작용으로 대체)
tooltip: tap → 표시 유지

탭 간 이동:
  주 네비게이션 — 탭 탭핑 (discoverability 기본)
  액셀러레이터 — 수평 스와이프 (탭 간 보조 이동 수단, 주 네비 아님)
  페이지 인디케이터 — 없음 (상단 탭바 언더라인이 위치 표시 역할)

FAB 위치:
  패널 우하단 44px (엄지 홈 포지션, 오른손 엄지 최적)
  context-aware: 현재 탭에 따라 할일/메모/일정 생성 모달 호출
```

## U12. 대칭 설계 원칙 ⭐

```
완료 축과 삭제 축은 같은 3층 회수 동선을 공유한다:
  체크 (완료) → 최근 완료 24h → 지난 완료 (영구 보관)
  스와이프 (삭제) → 최근 삭제 24h → 휴지통 (30일 hard delete)

같은 자리에서 대칭 노출:
  리스트 하단: "최근 완료 N개 →" / "최근 삭제 N개 →" 병행 배치
  탭바 메뉴: "기록" 한 진입점 → 2탭 모달 [완료] / [휴지통]

보관 기간 비대칭:
  완료 = 영구 보관 (업무 기록 · 레퍼런스 가치)
  휴지통 = 30일 후 hard delete (버린 것)

배후 철학:
  사용자가 "내가 뭘 한 거지" 혼동 없이 녹색 축(완료) / 분홍 축(삭제) 중 하나를
  시각으로 따라갈 수 있음. 축은 서로 오염되지 않음 (완료된 것이 삭제 회수로,
  삭제된 것이 완료 회수로 섞이지 않음).
```
