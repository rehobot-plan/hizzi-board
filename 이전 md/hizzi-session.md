## 협업 방식

```
나 (오너)          Claude.ai (여기)        Claude Code (VS Code)
──────────────────────────────────────────────────────────
방향 제시     →   설계 + 명령어 작성   →   실행 + 빌드 + 배포
```

### 핵심 원칙
- 나는 방향만 이야기한다
- Claude.ai가 파일 먼저 분석 → 완성본 코드 직접 작성 → Claude Code에 전달
- 추측으로 명령 금지 — 반드시 파일 내용 확인 후 정확한 코드 지정
- 에러 체크 필요한 작업은 하나씩, 결과 확인 후 다음
- 간단하고 연관된 작업은 묶어서 처리 가능
- 패치 3회 실패 → 파일 완전 새로 작성으로 전환
- Claude Code 메모리 부족 시 → VS Code 터미널 직접 사용
- 세션 마무리는 오너가 제안한다 (Claude가 먼저 마무리 제안 금지)
- Claude는 항상 일을 계속 진행시키는 태도를 유지한다

### 중간 요청 처리 원칙
```
방금 작업과 연관된 요청  → 즉시 반영
기존 리스트와 연관       → 해당 작업에 합쳐서 처리
완전히 새로운 요청       → Claude.ai가 우선순위 판단 후 리스트 배치
순서 변경 발생 시        → 오너에게 보고
```

### 매 작업 후 보고 형식
```
✅ 완료: [방금 한 것]
🔄 진행 중: [지금 할 것]
📋 다음 순서: [우선순위 순]
```

## Claude Code 명령어 패턴

### 파일 내용 확인 (수정 전 필수)
```
[파일명] 전체 내용을 채팅창에 텍스트로 붙여넣어줘. 코드블록으로.
```

### 표준 배포 명령어
```
Remove-Item -Recurse -Force .next
npm run build
git add . && git commit -m "메시지" && npx vercel --prod
완료되면 URL 한 줄 보고해줘.
```

### 메모리 부족 시 빌드
```
$env:NODE_OPTIONS="--max-old-space-size=4096"; npm run build
```

### git 파일 복구
```
git checkout src/components/[파일명].tsx
```
