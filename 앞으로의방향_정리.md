```

   - 명령: "LeaveManager.tsx에서 열람자도 관리자와 동일한 표 보이게.

   - 클릭 시 showLeave 상태로 메인 레이아웃 유지하면서 전환
   - 명령: "page.tsx 사이드바에 HIZZI BOARD 홈버튼 + 연차 메뉴 버튼 추가"
```

## 2. Hizzi Board에 AI 채팅 추가 (통합 계획)

### 방향 확정
- Rehobot을 별도 앱이 아닌 **히찌보드에 통합**
- 사이드바에 AI 채팅 패널 추가
- 게시판 보면서 AI 동시 사용 가능

### UI 구조
```
[게시판 영역] | [AI 채팅 패널]
              ← 토글 버튼으로 열고 닫기
```

### AI 채팅 권한 연동
```
일반 사용자 AI
├── 본인 패널 게시물 작성/수정/삭제
├── 연차 신청/조회
├── 달력 일정 추가
└── 공개된 게시물 조회

관리자 AI
├── 위 모든 기능
├── 전체 연차 조회
├── 공지사항 관리
└── 패널 관리
```

### 개발 순서
```
1단계: 히찌보드 현재 기능 완전 안정화
2단계: 사이드바에 AI 채팅 패널 추가 (웹)
3단계: Claude API 연동 + 권한 연동
4단계: 웹 완전 안정화 확인
5단계: 모바일 PWA 최적화
```

## 3. Rehobot Plan — 독립 앱 (별도 개발)

### 개요
- **앱 이름:** Rehobot Plan
- **슬로건:** "하지 않아도 됩니다. Rehobot이 합니다."
- **목적:** AI 대화로 일정을 말하면 달력+할일 자동 정리
- **작업 경로:** D:\Dropbox\Dropbox\Rehobot\rehobot-plan

### 히찌보드와의 관계
```
히찌보드 = 팀 협업 (내부용)
Rehobot  = 개인 스케줄러 (외부 상용화)
```

### 기술 스택
```
Next.js 14 + TypeScript + Tailwind
Firebase (Auth + Firestore)
Anthropic Claude API
모바일: 반응형 웹 → PWA → React Native 단계적 전환
배포: Vercel
```

### 핵심 기능
```
1. AI 대화창 (ChatGPT 스타일)
   사용자: "다음주 월요일 오후 2시 거래처 미팅"
   AI: "📅 4/6(월) 14:00 거래처 미팅 등록할까요?"
   [확인] → 달력 자동 등록

2. 달력 — 등록된 일정 표시
3. 할일 목록 — 마감일 정렬
4. 스마트 쿼리 — "이번주 일정 요약해줘"
```

### GTM 전략
```
1단계: 히찌보드 팀이 첫 번째 세팅 케이스
2단계: 지인 세팅 (1~10명)
3단계: 콘텐츠 — "비전문가가 AI 시스템 만드는 과정" 숏폼
4단계: 바이럴 (50~100명)
```
